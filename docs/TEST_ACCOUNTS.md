# 👤 Тестовые учётные записи

> В этом документе не должны храниться реальные пароли, API-ключи, токены или proxy credentials.
> Для локальной демонстрации создавайте пользователей через UI/API или через отдельный seed-скрипт с placeholder-значениями.

## 📋 ОБЗОР

Рекомендуемые типы учётных записей для тестирования:

| Тип | Email-пример | Пароль | Описание |
|-----|--------------|--------|----------|
| **Администратор** | `admin@example.local` | задаётся локально | Полный доступ к admin-only функциям |
| **Тестовый продавец** | `seller@example.local` | задаётся локально | Аккаунт с пользовательскими товарами/ключами |
| **Обычный пользователь** | любой локальный email | задаётся при регистрации | Пустой аккаунт для проверки onboarding |

## 🔐 АДМИНИСТРАТОР

### Данные для входа

Создайте администратора локально и сохраните пароль только в локальном password manager или временном seed-скрипте вне репозитория.

### Права доступа:
| Функция | Доступ | API ключ нужен? |
|---------|--------|-----------------|
| **Парсинг по URL** | ✅ Полный | ❌ Нет |
| **Мои товары (API)** | зависит от marketplace keys | ✅ Да |
| **Просмотр товаров** | только свои данные или admin views | ❌ Нет |
| **Добавление ключей** | ✅ Может добавлять | - |
| **Analytics** | ✅ Полный доступ | ❌ Нет |
| **Repricing** | ✅ Тестовый режим | ❌ Нет |
| **Billing** | ✅ Просмотр | ❌ Нет |
| **Notifications** | ✅ Все уведомления | ❌ Нет |
| **Users** | ✅ Управление | ❌ Нет |

### Особенности:
- Не должен обходить проверки API-ключей.
- Может выполнять admin-only действия, например управление глобальными AI settings.
- Не используйте общий публичный пароль для демонстраций.

## 👤 ОБЫЧНЫЙ ПОЛЬЗОВАТЕЛЬ

```
Email: любой (например, new@user.com)
Пароль: любой (минимум 6 символов)
```

### Права доступа:
| Функция | Доступ | API ключ нужен? |
|---------|--------|-----------------|
| **Парсинг по URL** | ✅ Полный | ❌ Нет |
| **Мои товары (API)** | ❌ Нет ключа | ✅ Требуется |
| **Просмотр товаров** | ✅ Только добавленные | ❌ Нет |
| **Добавление ключей** | ✅ Может добавлять | - |

### Особенности:
- Пустой аккаунт без данных
- Должен сам добавить API ключи
- Подходит для теста регистрации

## 🧪 СЦЕНАРИИ ТЕСТИРОВАНИЯ

### Сценарий 1: Тест парсинга (Админ)
```
1. Войти как admin@megashark.ai
2. Перейти на /parsing
3. Режим: "Конкуренты (URL)"
4. Вставить: https://www.wildberries.ru/catalog/24319630/detail.aspx
5. Нажать "Спарсить"
6. ✅ Результат: данные товара
```

### Сценарий 2: Тест API товаров (Админ)
```
1. Войти как admin@megashark.ai
2. Перейти на /parsing
3. Режим: "Мои товары (API)"
4. Выбрать: Wildberries
5. Нажать "Получить данные"
6. ✅ Результат: 2 демо-товара
```

### Сценарий 3: Полный аккаунт (Тестовый продавец)
```
1. Войти как test.seller@megashark.ai
2. Перейти на /profile → "Подключённые маркетплейсы"
3. ✅ Результат: 4 подключенных ключа

4. Перейти на /parsing → "Мои товары (API)"
5. Выбрать: Wildberries
6. Нажать "Получить данные"
7. ✅ Результат: товары через API (или демо)

8. Перейти на /products
9. ✅ Результат: 7 товаров в списке
```

### Сценарий 4: Регистрация нового пользователя
```
1. Перейти на /register
2. Заполнить данные (3 шага)
3. Выбрать маркетплейсы
4. Ввести тестовые ключи
5. ✅ Результат: аккаунт создан, ключи добавлены
```

## 🚀 БЫСТРЫЙ СТАРТ

**Для полного теста системы:**

1. **Войди как локальный тестовый продавец:**
   - Email: созданный локально
   - Пароль: заданный локально

2. **Пройди по всем разделам:**
   - `/dashboard` — главная панель
   - `/products` — товары (7 шт)
   - `/parsing` — парсинг (URL + API)
   - `/profile` — ключи маркетплейсов (4 шт)
   - `/billing` — подписка
   - `/notifications` — уведомления

3. **Для теста админ-функций:**
   - Выйди из аккаунта
   - Войди как локальный admin-пользователь
   - Попробуй "Мои товары (API)" — демо режим

---

**Важно:** аккаунты и пароли создаются локально. Не публикуйте реальные credentials в документации.
